### 명세서 1: 키워드 분류형

#### 장점 ✅

1. **문장 다양성 극대화**
   - 기준별로 7개 그룹 × 7개 기준 = 최대 49개 문장 조합
   - 공정 특성에 맞는 정확한 설명 가능
   - 예: "욕실" 공정 + "아이 안전" 기준 = "미끄럼·턱·넘어짐 위험을 우선 낮추는 공정입니다."

2. **확장성 우수**
   - 새로운 공정 추가 시 키워드만 추가하면 자동 분류
   - 그룹별 템플릿 추가로 문장 다양성 확장 용이

3. **정확성 높음**
   - 공정 특성과 기준을 모두 반영
   - GENERIC fallback으로 안전성 보장

#### 단점 ❌

1. **구현 복잡도 높음**
   - 키워드 매칭 로직 필요
   - 그룹 분류 함수 추가 구현 필요
   - 템플릿 관리 부담

2. **유지보수 부담**
   - 키워드 목록 관리 필요
   - 그룹 정의 변경 시 영향 범위 큼
   - 테스트 케이스 증가

3. **과도한 세분화 가능성**
   - 모든 공정이 그룹에 매칭되지 않을 수 있음
   - GENERIC 사용 빈도가 높아질 수 있음
